## Tone and voice

### Words to use

Direct, specific, number-forward, factual. Prefer verbs over adjectives. Prefer nouns over metaphors. When you describe an action, say what it does, not what it feels like. Sparse, precise, active voice. One sentence per idea. No filler. Lists over paragraphs. Numbers over adjectives.

### Words to avoid

Hype words ("revolutionary", "game-changing", "unleash", "supercharge", "unlock"), emojis, exclamation marks, overpromising. Effort-words ("simply", "just", "easy", "powerful", "seamless") — they describe the effort, not the outcome. Corporate verbs ("utilize", "leverage", "synergy"). Filler ("kindly", "please note that").

### Voice examples

> Supplied 1,000 USDC at 4.2% APY. Withdraw anytime — no lockup.

> Above-threshold actions route to Telegram for human approval.

> 42 ms. 200 MB. 0 errors.

## Usage dos and don'ts

**Do:**

- Use the shadcn CSS variables (`bg-background`, `text-foreground`, `bg-primary`) everywhere. Never hardcode hex.
- Keep the 4 px spacing grid (`gap-2`, `p-4`, `px-6`).
- Respect `prefers-color-scheme` by using the `.dark` class on `<html>` based on the system setting.
- Use `font-mono tabular-nums` for any number that updates in place (balances, prices, counters, treasury addresses).
- Treat cyan (`--primary`) as the *single* accent. Reserve it for primary CTAs, focus rings, status confirmations.
- Test every component in both light and dark mode before considering it done.

**Don't:**

- Hardcode colors (`#FF0000`, `rgb(...)`) in component files.
- Use `transition: all` — always specify `transition-colors`, `transition-transform`, etc.
- Introduce a second brand color. The Inference palette is monochrome + cyan by design; a second hue dilutes the brand.
- Override shadcn tokens per-component — if a color is wrong, fix it in `globals.css` once.
- Add gradients without re-running `brand-design`. Free-handed gradients break the minimal mood.
